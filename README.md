## О проекте

Данный проект представляет собой простую HTML-страницу с вызовом методов GREEN-API и минимальный backend на Go для проксирования запросов к API.

### Использованные API-методы GREEN-API

В проекте используются следующие методы:

- [getSettings](https://green-api.com/en/docs/api/account/GetSettings/)
- [getStateInstance](https://green-api.com/en/docs/api/account/GetStateInstance/)
- [sendMessage](https://green-api.com/en/docs/api/sending/SendMessage/)
- [sendFileByUrl](https://green-api.com/en/docs/api/sending/SendFileByUrl/)

Структура запросов и порядок параметров в URL реализованы в соответствии с документацией GREEN-API.

### Почему не использовался готовый SDK

Для ускорения выполнения тестового задания готовые Golang-пакеты GREEN-API SDK не использовались.

Это было сделано намеренно, чтобы:
- быстрее собрать рабочее решение;
- сократить время на изучение дополнительных абстракций SDK;
- реализовать только необходимый минимум функциональности по требованиям задания.

### sendFileByUrl

Метод `sendFileByUrl` требует обязательный параметр `fileName` вместе с `chatId` и `urlFile`.

Чтобы упростить интерфейс страницы и не добавлять лишнее поле ввода, параметр `fileName` в проекте реализован как заглушка по умолчанию.

При необходимости его можно заменить на вводимое значение из формы.

### Итоговая реализация

Итоговая версия проекта собрана в виде бинарника, скомпилированного командой `go build`, и запускается как отдельный сервис за reverse proxy на NGINX.

Доступ к приложению ограничен basic authentication на уровне NGINX, а для HTTPS используется самоподписанный SSL-сертификат.

Такой вариант был выбран для упрощения публикации и защиты входа на демонстрационную страницу.
